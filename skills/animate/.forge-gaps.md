## C. New rules to add (synthesised from sources + deductions)

1. **Skill('X') means LITERAL invocation.** When the skill text says "invoke /impeccable", the runner must use the Skill tool, not Read the SKILL.md.
2. **Per-role motion recipe table.** A short matrix mapping `route role` → typical proposals (read from web-animations route-roles.md).
3. **Removal proposals must cascade.** If a Proposal removes a motion.X element, the skill must scan for orphaned imports/hooks/state and remove them too.
4. **ARIA-state replacement check.** When a visual state indicator (color fill, motion sweep) is removed, the proposal must include the ARIA equivalent (aria-pressed, aria-current, role=switch, aria-expanded).
5. **Floor citation requirement.** Every motion-threshold claim (scale delta, duration, y-offset) cites the floor-rules.md rule it satisfies.
6. **No invented edge cases.** If the target route doesn't have a form, the artifact says "no form on this route — N/A". Don't fabricate proposals for absent features.
7. **One artifact = one target.** A single `/animate` invocation produces edits for ONE target page or feature. The rubric should not penalise narrow scope.
