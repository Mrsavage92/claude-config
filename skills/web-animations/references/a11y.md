# Accessibility — Beyond Reduced Motion

`prefers-reduced-motion` is necessary, not sufficient. Full a11y coverage for motion has 8 layers.

Sources:
- WCAG 2.2 SC 2.3.3 — Animation from Interactions
- WCAG 2.2 SC 2.2.2 — Pause, Stop, Hide
- WebKit "Responsive Design for Motion" (six vestibular triggers — see `floor-rules.md`)
- W3C Inclusive Design — Cognitive a11y

---

## 1. Vestibular safety

Reduced motion ALWAYS disables: parallax, autoplay video, infinite spinning, large-area motion, looping anims, `filter: blur()` transitions.

Tier 3 features (`PinnedSection`, `SmoothScroll`, `CardStack`, `CursorParallax`) MUST early-return on `useReducedMotion()` — every kit component does this. See `floor-rules.md` Floor 3.

Test: toggle Settings → Display → Reduce motion (macOS) or Animations off (Windows) and reload the page.

---

## 2. Focus management on route/state transitions

`AnimatePresence mode="wait"` exits old content before new mounts → focus is lost.

Move focus to the new page's `<h1>` (or first interactive element) on transition complete:

```tsx
const headingRef = useRef<HTMLHeadingElement>(null)
useEffect(() => {
  headingRef.current?.focus()
}, [pathname])
return <h1 ref={headingRef} tabIndex={-1}>...</h1>
```

For View Transitions API: wire focus restoration into the `ready` promise.

```ts
const t = document.startViewTransition(() => updateDOM())
t.ready.then(() => document.querySelector('h1')?.focus())
```

---

## 3. Screen reader announcements

Number tickers, count-ups, value changes: WRAP visible content in `aria-hidden`, expose final value via `aria-label` on the parent. Kit `NumberTicker` does this:

```tsx
<span aria-label="1,234">
  <motion.span aria-hidden>{display}</motion.span>
</span>
```

Live regions (`aria-live="polite"`) only for legitimate status changes (notifications, errors), NOT for decorative animation triggers — gets verbose fast and screen readers will throttle/skip.

---

## 4. Keyboard parity for hover-driven animations

Anything triggered by `onMouseMove` (`MagneticButton`, `CursorParallax`, `TiltCard`) MUST have a keyboard equivalent or graceful fallback.

Kit components use `(hover: hover)` media query — touch/keyboard users see the static version automatically.

Do NOT rely on `:focus` to replicate `:hover` motion — the mouse-position math doesn't apply.

---

## 5. SplitText reveals

- Set `aria-label="full text"` on the parent
- Set `aria-hidden` on each split word/character so screen readers don't read "H. e. l. l. o."
- Kit `SplitReveal` handles both:
  ```tsx
  <Tag aria-label={text}>
    {parts.map((p, i) => <motion.span key={i} aria-hidden ...>{p}</motion.span>)}
  </Tag>
  ```
- Test with VoiceOver (macOS), NVDA (Windows), or Chrome Lighthouse a11y audit

---

## 6. Modal focus traps

Kit `AnimatedModal` handles Escape, but YOU must trap Tab inside the modal and restore focus on close.

Use `react-focus-lock` or `@radix-ui/react-focus-scope` — don't hand-roll a focus trap, edge cases are brutal.

Restore focus to the trigger element on close:

```tsx
function ModalTrigger() {
  const triggerRef = useRef<HTMLButtonElement>(null)
  const [open, setOpen] = useState(false)
  
  function handleClose() {
    setOpen(false)
    setTimeout(() => triggerRef.current?.focus(), 0)
  }
  
  return (
    <>
      <button ref={triggerRef} onClick={() => setOpen(true)}>Open</button>
      <AnimatedModal open={open} onClose={handleClose}>
        <FocusLock>{/* content */}</FocusLock>
      </AnimatedModal>
    </>
  )
}
```

## 7. Tab order doesn't change with animations

An element being scaled, rotated, or off-stage visually is STILL in tab order.

Hide truly-removed elements with `aria-hidden="true"` + `tabIndex={-1}`, OR remove from DOM entirely via `AnimatePresence` exit.

`opacity: 0` does NOT remove from tab order. `visibility: hidden` does, but `pointer-events: none` does not.

---

## 8. Pause / Stop / Hide (WCAG 2.2.2)

Any animation that:
- Lasts longer than 5 seconds
- Plays automatically (no user trigger)
- Plays in parallel with other content

…must provide a way to pause, stop, or hide it.

The kit's `Marquee` runs infinitely. Add a visible pause control if the marquee is on a content page (vs a logo strip below the fold). The current kit does not provide this — consumer's responsibility.

Recommended pattern:

```tsx
const [paused, setPaused] = useState(false)
<Marquee paused={paused} ... />
<button onClick={() => setPaused(!paused)} aria-label={paused ? 'Resume marquee' : 'Pause marquee'}>
  {paused ? 'Resume' : 'Pause'}
</button>
```

Kit `Marquee` does not yet accept a `paused` prop. Open follow-up: add `pause-on-hover` and explicit `paused` prop.

## Test matrix

| Layer | Tool | Pass criterion |
|---|---|---|
| 1 Vestibular | Settings toggle + page reload | All Tier 3 components substitute or skip |
| 2 Focus on route | Manual keyboard nav across routes | First `<h1>` receives focus on transition end |
| 3 Screen reader | NVDA / VoiceOver smoke test on hero + ticker | Hero text read once, ticker reads final value not increments |
| 4 Keyboard parity | Tab through `MagneticButton`, `TiltCard` | All actionable; no motion required to interact |
| 5 Split text | Screen reader on `SplitReveal` | Full text read once, not letter-by-letter |
| 6 Modal trap | Tab inside open modal | Tab cycles within modal; Escape closes; focus returns to trigger |
| 7 Tab order | Tab through page with hidden elements | No tabstops on `aria-hidden` or removed elements |
| 8 Pause/Stop | Inspect Marquee + autoplay video | Each has a pause control if > 5s or autoplay |

## CI integration

Playwright + axe-core:

```ts
import { test, expect } from '@playwright/test'
import AxeBuilder from '@axe-core/playwright'

test('reduced motion compliance', async ({ page }) => {
  await page.emulateMedia({ reducedMotion: 'reduce' })
  await page.goto('/')
  
  const results = await new AxeBuilder({ page }).analyze()
  expect(results.violations).toEqual([])
  
  // No running animations after load under reduce
  const running = await page.evaluate(() =>
    document.getAnimations().filter(a => a.playState === 'running').length
  )
  expect(running).toBe(0)
})
```

Add this to every kit-consuming project's E2E suite.
