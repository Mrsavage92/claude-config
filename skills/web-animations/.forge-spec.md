# Forge Spec — /web-animations

**Captured**: 2026-05-17
**Mode**: forge
**Target score**: 90/100

---

## What the skill is supposed to produce

A site-type-adaptive motion system for web-* skills: an importable component kit (Tier 1–3), a SKILL.md decision matrix that picks motion intensity based on the target site's actual purpose, and a grader script that audits downstream projects for tier markers and reduced-motion compliance.

---

## What broke (the trigger for this forge)

**Concrete failure**:
- Input run: the skill currently exists and downstream skills (web-scaffold, web-page, web-evolve, polish, animate) reference it
- Output that came back: a 4-tier reference + kit components, but the rubric for which tier applies to which site is shallow — only distinguishes "SaaS / Marketing / Brand / Docs"
- Why it was wrong: it does not account for local service businesses, professional services, ecommerce, content/publication sites, dashboards, portfolios, or nonprofits. Defaults bias toward Marketing/Brand "signature moment" thinking which is wrong for many real sites. Pairs with the /web-evolve adaptive redesign plan the user laid out: skills must classify the site first, then choose the path.
- Where it was scored differently: in this same conversation, my own honest re-score went from 88 → 65 once I stopped grading my own prose and looked at gaps. Pairs with [[feedback_no_self_quality_claims]].

## Success criteria (each verifiable from the artifact alone)

1. **Site-type classification matrix** — the artifact lists ≥10 site types (Local Service, Pro Services / Agency, SaaS, Ecommerce, Marketplace, Portfolio, Content/Publication, Nonprofit, Event/Venue, App Dashboard, Documentation, Hybrid) with motion ceiling, floor and recommended tier per type.
2. **Adaptive Tier 3 gating** — Tier 3 signature moves are NOT recommended for Local Service, Documentation, Dashboard, or Content/Publication. The artifact spells this out.
3. **Importable kit** — `kit/` directory exposes ≥14 components, every component carries a `// web-animations: Tier N` marker, the kit's grader exits 0 on its own files.
4. **Reduced-motion respect** — every Tier 2+ component in the kit either short-circuits on `useReducedMotion()` OR delegates to a static branch. Verifiable by grep.
5. **Bundle budget table** — actual gzipped sizes (motion, gsap, lenis, three, rive, lottie) and a rule for when to lazy-load Tier 3 libraries.
6. **Grader produces structured JSON** — running `node grader/audit-animations.mjs <project>` outputs JSON with tierBreakdown, unmarkedMotionFiles, tier3FilesMissingReducedMotionGuard, plus non-zero exit on violations.
7. **No banned phrases** in SKILL.md: `comprehensive, robust, production-ready, world-class, premium, perfect, best-in-class, enterprise-grade, battle-tested, deeply, holistic, seamless, cutting-edge`.
8. **Skill routing matrix** — a clear "when to use this vs `/animate` vs `/polish` vs `/overdrive` vs `/style-mirror`" section. No overlap left undefined.
9. **Anti-patterns list** — ≥6 ship-blockers tied to real failure modes, not generic advice.

## Constraints

- **Length cap**: SKILL.md ≤ 600 lines; heavier content in `references/` or `kit/`
- **Dependencies available**: motion v12, gsap 3.12+, lenis 1.1+, vitest, playwright, Node 20+ for grader
- **Dependencies unavailable**: no Anthropic SDK in the kit, no MCP requirement in the grader
- **Must-haves locked**: kit + grader stay where they are at `~/.claude/skills/web-animations/{kit,grader}/`; SKILL.md references them
- **Banned approaches**: "make it like Bruno Simon" defaults, forcing dashboards into Tier 3, suggesting motion on documentation/content sites, custom cursors as default (per [[feedback_no_custom_cursor_by_default]])

---

## Edge cases to cover (artifact must demonstrate)

- Local service business landing page — what tier ceiling? what's banned?
- SaaS product dashboard route — different from SaaS marketing route?
- Documentation site (high text density, very little motion)
- Ecommerce product detail page (motion serves purchase confidence, not awe)
- Portfolio site (motion CAN signal craft, but only specific moves)
- Nonprofit/cause site (motion must not distract from message)
- Hybrid site (one part marketing, one part app) — per-route classification
- Touch-only device (no `(hover: hover)`) — every kit Tier 2/3 component handles gracefully
- prefers-reduced-motion enabled — Playwright spec must pass
- Bundle budget blown by Tier 3 libs — artifact recommends lazy-loading recipe

## Score-down triggers (user-defined deductions, beyond rubric)

If any of these appear in the artifact, treat as blocking_issue regardless of dimension scores:
- "Premium SaaS" / "Awwwards" / "SOTM" / "Bruno Simon" framing as default direction
- Recommending Tier 3 to local service or documentation sites without an explicit override
- Inline snippets in SKILL.md duplicating kit components (the kit is the source of truth)
- Grader that grades only on motion library presence (must check tier markers + reduced-motion guards)
- Any self-flattery sentence like "now produces a polished motion system"
